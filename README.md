# EcoMarketShop

> 一款完全基于**服务端**运行的 Fabric 模组，集**经济系统、管理员商店、全球市场**于一体。玩家**无需安装任何客户端模组**，通过原版箱子 GUI 即可体验完整的交易系统。

## 📖 项目简介

EcoMarketShop 是面向 Minecraft 1.21.8 Fabric 服务端的经济套件。所有扣款、发物、校验逻辑均在服务端权威执行，客户端仅作为显示界面，从根本上杜绝了客户端作弊的可能。模组采用「GUI 点击 → GUI 确认界面」的核心交互模式，所有交易操作均通过服务端箱子 GUI 确认后执行，有效防止误触操作。

## ✨ 核心功能

- 💰 **经济系统**：玩家拥有独立的 Coin 余额（整数），数据以 JSON 文件持久化存储，支持 OP 后台查询与修改。
- 🏪 **管理员商店**：OP 通过配置文件自定义无限库存商品，玩家在 GUI 中左键点击商品购买。可堆叠物品左键直接购买 1 个，不可堆叠物品弹出 GUI 确认界面；右键购买一组，Shift+左键打开数量选择 GUI。商店为单向（仅购买）。
- 🌍 **全球市场（拍卖行）**：玩家自由交易的场所。卖家可将背包物品拖拽至 GUI 上架并设置单价；买家分页浏览、整单购买。成交时系统自动从卖家收入中扣除 5% 手续费。
- 🖥️ **纯原版 GUI**：基于箱子界面（`GenericContainer`），无需任何客户端资源包或模组。
- 🔒 **安全机制**：内置操作冷却 CD（防高频刷物），所有交易由服务端严格校验。

## 📋 环境要求

| 组件 | 版本要求 |

## 🚀 安装与配置

### 安装步骤

1. 确保服务端为 **Fabric 1.21.8**，并已安装 **Fabric API**。
2. 将本模组（`EcoMarketShop-1.0.0.jar`）放入服务端的 `mods/` 文件夹。
3. 启动服务器，模组将自动在 `./config/` 目录下生成配置文件。
4. 按需修改 `config/shop_config.json` 后，执行 `/market reload` 或重启服务器使其生效。

### 从源码构建

```bash
# Windows
gradlew.bat build

# macOS / Linux
./gradlew build
```

构建产物位于 `build/libs/` 目录下。

### 配置文件

模组启动后会在 `./config/` 目录下生成以下 JSON 文件：

| 文件 | 说明 |
|------|------|
| `economy_config.json` | 经济配置（货币名、初始余额、手续费率等） |
| `shop_config.json` | 管理员商店商品配置（修改后需 `/market reload` 生效） |
| `market_data.json` | 市场挂单数据（自动生成，请勿手动编辑） |

## 📝 命令列表

| 命令 | 权限 | 描述 |
| :--- | :--- | :--- |
| `/eco` | 玩家 | 查询自己的当前余额 |
| `/eco set <玩家名> <数量>` | OP | 设置指定玩家的余额 |
| `/eco add <玩家名> <数量>` | OP | 增加指定玩家的余额 |
| `/eco reduce <玩家名> <数量>` | OP | 扣减指定玩家的余额（余额不足拒绝） |
| `/eco view <玩家名>` | OP | 查看指定玩家的余额 |
| `/shop` | 玩家 | 打开管理员商店 GUI（仅购买） |
| `/market` | 玩家 | 打开全球市场浏览与购买 GUI |
| `/market sell <单价>` | 玩家 | 打开市场上架 GUI |
| `/market my` | 玩家 | 打开我的挂单 GUI（可下架自己的挂单） |
| `/market reload` | OP | 热重载市场/商店配置文件 |

> 所有 GUI 内的操作均为**左键单击**触发，涉及交易的操作会弹出 GUI 确认界面，点击 **[✓ 确认]/[✗ 取消]** 按钮完成操作。
